**1. システム概要**

本システムは、ダイビングショップにおける予約受付・問診収集・顧客情報管理を、Google
フォームと Google
スプレッドシートを組み合わせて実現する。RDBMSを使用せず、スプレッドシートをデータストアとして活用することで、低コストかつ即時導入を可能にする。

**1.1 システム構成**

  --------------------------------------------------------------------------------------
  **コンポーネント**   **ツール**                       **役割**
  -------------------- -------------------------------- --------------------------------
  入力フォーム         Google フォーム（予約）          顧客からの予約・問診情報の受付

  データストア         Google スプレッドシート          全データの一元管理・閲覧

  自動連携             フォーム回答の自動転記           回答 → シートへ自動追記

  管理・集計           スプレッドシート関数・フィルタ   スタッフによる予約管理・確認

  通知（任意）         Google Apps Script               メール通知・重複チェック
  --------------------------------------------------------------------------------------

**1.2 管理対象データ**

- 顧客情報（氏名・連絡先・緊急連絡先・資格・経験等）

- 予約情報（日程・コース・人数・ステータス・担当スタッフ）

- 問診情報（健康状態・既往歴・同意確認）

- マスタ情報（コース・スタッフ・ステータス）

**2. データフロー概要**

顧客がフォームから予約・問診を送信すると、各シートに自動転記される。スタッフはスプレッドシートを開いて確認・管理を行う。

+:---------------------------------------------------------------------:+
| **【データフロー】**                                                  |
|                                                                       |
| ① 顧客 → Google フォーム（予約フォーム / 問診フォーム）で入力・送信   |
|                                                                       |
| ↓ フォーム回答の自動転記（Google スプレッドシート連携）               |
|                                                                       |
| ② 予約シート（Reservations）に1行追記                                 |
|                                                                       |
| 問診シート（Medical Questionnaire）に1行追記                          |
|                                                                       |
| ↓ スタッフが手動または GAS で顧客台帳と照合                           |
|                                                                       |
| ③ 顧客台帳（Customers）に顧客情報を登録・更新                         |
|                                                                       |
| ④ スタッフ → スプレッドシートで予約確認・ステータス更新・担当割当     |
+-----------------------------------------------------------------------+

**5. シート間リレーション**

各シートはキー列によって論理的に結合される。スプレッドシートでは VLOOKUP
/ INDEX・MATCH 関数でデータを参照する。

  --------------------------------------------------------------------------------------------
   **参照元シート**     **参照列**     **参照先シート**     **参照列**    **取得内容**
  ------------------ ---------------- ------------------ ---------------- --------------------
       予約一覧      **顧客ID (FK)**       顧客台帳      **顧客ID (PK)**  氏名・電話・メール

       予約一覧         **コースID       コースマスタ       **コースID    コース名・単価
                          (FK)**                              (PK)**      

       予約一覧        **スタッフID     スタッフマスタ     **スタッフID   スタッフ氏名
                          (FK)**                              (PK)**      

       予約一覧       **ステータスID   ステータスマスタ   **ステータスID  ステータス名
                          (FK)**                              (PK)**      

       問診回答      **予約ID (FK)**       予約一覧      **予約ID (PK)**  顧客ID・ダイブ日

       問診回答      **顧客ID (FK)**       顧客台帳      **顧客ID (PK)**  氏名（照合用）
  --------------------------------------------------------------------------------------------

> *※
> スプレッドシートはRDBのJOINではなくVLOOKUP/INDEX-MATCHで参照するため、参照元の値変更時は数式再計算が自動で行われる。*
>
> *※
> キーが変わった場合は連鎖的に参照も更新されるため、一度登録したIDは変更しないこと。*

**6. Google フォーム連携設計**

**6.1 予約フォーム（→ 予約一覧シートへ自動転記）**

     3     メールアドレス               メール形式       ◎ 必須   メールアドレス（台帳照合キー）

**9. 運用ルール・注意事項**

**9.1 ID管理ルール**

- 各シートのPK（主キー）IDは一度登録したら変更しない。

- 顧客IDはメールアドレスをユニークキーとして照合し、重複登録を防ぐ。

- IDの採番はGAS（onFormSubmit /
  手動トリガー）で行うことを推奨。手動採番の場合は「採番台帳」列を別途設けること。

**9.2 データバックアップ**

- Google
  スプレッドシートはGoogleドライブ上にあるため自動バージョン管理（変更履歴）が有効。

- 月次で「ファイル →
  コピーを作成」でスナップショットを別フォルダに保存することを推奨。

- 問診・顧客台帳は特に重要な個人情報を含むため、四半期ごとにCSVエクスポートし暗号化バックアップを推奨。

**9.3 個人情報取り扱い**

- 顧客情報・問診情報は個人情報保護法の対象。取得目的の明示とプライバシーポリシーをフォームに掲示すること。

- スプレッドシートを外部公開・社外共有しないこと。スタッフへの共有はGoogleアカウント単位で管理。

- 免責同意の記録（問診シート）は法的証跡として保管。削除・改ざん禁止。

**9.4 スプレッドシートの限界と注意**

  **最大行数**                 10,000,000セル / スプレッドシート      年度ごとにファイルを分割する
